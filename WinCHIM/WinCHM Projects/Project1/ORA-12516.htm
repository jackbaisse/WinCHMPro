ORA-12516：TSN：监听程序找不到符合协议堆栈要求的可以处理程序

原因：一般是由于数据库的当前会话数不满足造成的，检查或者修改数据库两个参数processes和sessions 。

两者关系：sessions=(1.1*processes+5)

大概处理步骤：

1、通过sqlplus登录数据库，检查当前进程数

select count(*) from v$process;

2、检查当前会话数

select count(*) from V$session;

3、检查参数配置

show parameters  process;

show parameter session;

4、修改进程数和会话数两个参数

alter system set processes=500 scope=spfile;

alter system set sessions=555 scope=spfile;

5、重启数据库，使参数生效

shutdown abort;

startup;
