8、版本回退
1）git reset --hard HEAD^   --HEAD^ 表示上个版本 HEAD^^ 表示上上个版本
2）git reset --hard HEAD~100表示100个版本
3）git reset --hard HEAD commit id编号

9、查看每一次命令
git reflog

10、工作区文件与最新版本库区别
git diff HEAD -- 123.txt   -----文件名

11、丢弃工作区的修改
git checkout -- 123.txt
--两种状态
一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。 git reset --HEAD^

12、丢弃暂存区的修改
git reset HEAD <filename>
